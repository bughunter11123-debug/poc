🔓 Open Redirect PoC - Gameboost /otp/send-login

Vulnerability Details

Endpoint: GET /otp/send-login

Issue: The Referer header is reflected in redirects without validation

Impact: Attacker-controlled redirect via Referer header manipulation

⚠️ Attack Scenario

An attacker creates a page that automatically triggers the vulnerable endpoint with a malicious Referer header, causing users to be redirected to an attacker-controlled domain.

How This Works (No Burp/Proxy Needed)

  1. HTML form with referrerpolicy="unsafe-url" allows full control over Referer header
  2. JavaScript dynamically creates and submits requests with custom headers
  3. Browser naturally includes the Referer header in requests
  4. Server reflects the Referer without validation, causing redirect to attacker's URL
  5. User is redirected entirely within the browser - no manual proxy needed!

Real-World Attack Flow

1. Attacker sends phishing email with link to their malicious page
2. Victim clicks the link (this page or attacker's version)
3. Page automatically triggers: POST /otp/send-login
4. With Referer: https://attacker.evil/phishing/page
5. Server redirects to the Referer header value
6. Victim lands on attacker's fake login page
7. Credentials harvested ✓

Proof of Concept Code

<form action="https://gameboost.com/otp/send-login"
